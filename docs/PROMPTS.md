Lets build this CLI in Rust. Create the project in the rust-port folder. 
The CLI will be called token-vault-proxy (Auth0 Token Vault Proxy). Executable will be `tv-proxy`. It allows users to access third-party services (Gmail, Slack, etc.) on behalf of authenticated users via Auth0 Token Vault. Designed for both humans and AI agents. 
The CLI will support the following commands:
- `fetch <service> <url> [options]` - Same as the fetch command in the auth0-tv CLI. It will be a proxy command that can be used by AI agents to make authenticated API calls to third-party services. The CLI will fetch the token from Auth0 Token Vault and make the API call on behalf of the user and return the response as JSON. The CLI will also support passing additional options such as HTTP method, headers, body etc.
- `connect <provider> [options]` - same as the connect command in the auth0-tv CLI. But instead of connecting services like gmail and calender, it will be used to connect to oauth providers supported by auth0. For example 'google-oauth2', 'github', 'sign-in-with-slack' etc. Use friendly aliases like 'google', 'slack' as provider names and resolve to actual provider names in auth0. Support sending either actual name or friendly name. Have a registry of known providers, their aliases and default scopes (same as in current CLI). If scopes is provided as option, merge with default scopes and remove duplicates before sending to auth0. For unknown providers, pass provider name and scopes as is to auth0 and let it handle the error if provider is not found or scopes are invalid. Should support --allowed-domains option to specify allowed domains for the connection (same as in current CLI).
- `connections` - same as the disconnect command in the auth0-tv CLI.
- `disconnect <provider> [options]` - same as the disconnect command in the auth0-tv CLI.
- `login, logout, status` - same as the auth commands in the auth0-tv CLI.
- `init` - An init command that will help to install auth0 CLI, run configure-auth0-token-vault and set up everything needed to get started with the CLI. Then runs the login command. It will be an interactive command that will ask for necessary info and provide a final summary and instructions.
